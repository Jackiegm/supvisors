Overview

Supervisors is a control system for distributed applications over multiple Supervisor instances.

This piece of software is born from a common need in embedded systems where applications are distributed over several boards.
This problematic comes with the following challenges:

  • have a detailed status of the application,
  • have statistics about the resources taken by the application,
  • have a basic status of the boards,
  • start / stop processes dynamically,
  • distribute the same application according to different platforms,
  • control the application from outside of the platform,
  • secure the control / status interfaces,
  • deal with failures:
    • missing board when starting,
    • crash of a process,
    • crash of a board,
  • deal with loading (CPU, memory, etc).

Icing on the cake:

  • it should be free, open source, without export control,
  • it shouldn't require specific administration rights (root).

After some researches on the net, it seems that there is no simple, free and open source solution that meets all these requirements.
Supervisor can handle a part of these requirements but it only works on a single UNIX-like operating system.
The Supervisor website references some tools that deal with multiple Supervisor instances but they only consist in dashboards and none of them is focused on the application itself.
However, the extensibility of Supervisor made it possible to implement the missing requirements.

In this documentation, a Supervisors instance refers to a Supervisor instance including a Supervisors extension.
Supervisors is used to refererence the defined set of these instances.

Platform Requirements

Supervisors has been tested and is known to run on Linux (CentOS 7.2)

Supervisors will not run at all under any version of Windows.

Supervisors works with Python 2.7 or later but will not work under any version of Python 3.

Installation

Packaging

Temporary: get a copy of the supervisors directory from here and export to PYTHONPATH before running Supervisor

Supervisors has the following dependencies:

  • Supervisor: base software, extended by Supervisors
  • PyZMQ: used for TCP publish / subscribe pattern.
  • netifaces: used to get IPv4 aliases from host name.
  • lxml: for XSD validation of the XML deployment file(optional).

Running Supervisors

Same principle as Running Supervisor but using multiple UNIX-like operating systems.

Note that a script may be required to start Supervisor on several addresses if not configured to run automatically at startup (ssh loop for example).
All Supervisor instances should be started during a given lap of time (configuration is explained later) so that Supervisors works as expected.
